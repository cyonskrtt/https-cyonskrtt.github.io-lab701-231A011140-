Câu 1

Sự khác biệt giữa let, const và var trong JavaScript?

var có phạm vi function-scope, có thể khai báo lại và bị "hoisting" (đưa lên đầu) kèm giá trị undefined. let và const có phạm vi block-scope (chỉ tồn tại trong cặp {} chứa nó), không thể khai báo lại trong cùng scope. Khác biệt giữa let và const: biến khai báo bằng const không thể gán lại giá trị sau khi khởi tạo (riêng object/array vẫn có thể thay đổi nội dung bên trong).

// var: bị hoisting, function-scope console.log(a); // undefined var a = 10; // let: block-scope, có thể gán lại let b = 5; b = 8; // hợp lệ // const: không thể gán lại const c = 1; c = 2; // ❌ Lỗi: Assignment to constant variable
Câu 2

Box Model trong CSS là gì? Nêu thứ tự các thành phần.

Box Model mô tả cách trình duyệt tính toán kích thước và khoảng cách của mỗi phần tử HTML, gồm 4 lớp tính từ trong ra ngoài: content (nội dung) → padding (khoảng đệm bên trong) → border (viền) → margin (khoảng cách bên ngoài). Thuộc tính box-sizing: border-box giúp padding và border được tính vào trong chiều rộng/cao đã khai báo, tránh việc phần tử bị "phình" to hơn dự kiến.

.box { width: 200px; padding: 20px; border: 2px solid #39ff8e; margin: 10px; box-sizing: border-box; /* width tổng = 200px, không cộng dồn */ }
Câu 3

Sự khác nhau giữa Flexbox và Grid trong CSS?

Flexbox là hệ thống bố cục một chiều (one-dimensional), phù hợp để sắp xếp các phần tử theo hàng hoặc cột, tự co giãn theo không gian còn lại — thường dùng cho thanh menu, danh sách card theo hàng. Grid là hệ thống bố cục hai chiều (two-dimensional), cho phép kiểm soát đồng thời cả hàng và cột, phù hợp cho bố cục trang tổng thể (header, sidebar, content, footer) hoặc lưới sản phẩm phức tạp.

/* Flexbox: sắp xếp theo 1 chiều */ .menu { display: flex; gap: 12px; } /* Grid: sắp xếp theo lưới 2 chiều */ .layout { display: grid; grid-template-columns: 240px 1fr; grid-template-rows: 64px 1fr 80px; }
Câu 4

LocalStorage là gì? Khác gì với SessionStorage và Cookie?

LocalStorage là bộ nhớ lưu trữ dữ liệu dạng key-value ngay trên trình duyệt, dữ liệu không bị mất khi đóng tab hoặc tắt trình duyệt, dung lượng khoảng 5–10MB. SessionStorage hoạt động tương tự nhưng dữ liệu chỉ tồn tại trong phiên làm việc của tab đó, mất khi đóng tab. Cookie có dung lượng nhỏ hơn nhiều (~4KB), được gửi kèm theo mỗi request lên server, và có thể đặt thời gian hết hạn cụ thể — thường dùng cho phiên đăng nhập, xác thực.

// Lưu dữ liệu localStorage.setItem("user", JSON.stringify({ name: "Khải" })); // Đọc dữ liệu const user = JSON.parse(localStorage.getItem("user")); // Xoá dữ liệu localStorage.removeItem("user");
